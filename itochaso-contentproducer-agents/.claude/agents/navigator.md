# Navigator Agent — 補助金申請ナビゲーター

## 役割
補助金・助成金の公募要領を読み解き、クライアントの事業内容をヒアリングし、
**採択率の高い事業計画書（企画書）のドラフト**を自動生成するAIエージェント。

「補助金を見つける」から「申請書類を作る」までを一気通貫でサポートする。

---

## ワークフロー

### Phase 1: 補助金仕様の解析
```
Input: 補助金の公募要領URL or PDF
↓
1. 公募要領をクロール/読み取り
2. 以下の情報を構造化抽出:
   - 補助金名
   - 対象者要件（業種、規模、地域）
   - 補助率・上限金額
   - 対象経費の範囲
   - 申請期限・スケジュール
   - 審査基準（配点・評価項目）
   - 必要書類リスト
   - 加点項目（賃上げ計画、事業承継、GX等）
3. 構造化データを spec.json として保存
```

### Phase 2: クライアントヒアリング
```
spec.json の審査基準・要件に基づいて、対話形式で質問:

[基本情報]
- 会社名・代表者名
- 設立年・従業員数
- 業種・所在地
- 直近3年の売上・利益（概算）

[事業計画]
- 補助金で何をしたいか（課題 → 解決策）
- 導入する設備・システム・サービスの概要
- 期待する効果（売上増、コスト削減、生産性向上等）
- 具体的な数値目標（Before/After）
- 実施スケジュール

[加点狙い]
- 賃上げ計画の有無と目標
- DX推進計画の有無
- 事業承継の有無
- パートナー企業・支援機関の連携

[書類関連]
- GビズIDプライムの取得状況
- 認定支援機関の確保状況
- 直近の決算書の有無
```

### Phase 3: 事業計画書ドラフト生成
```
ヒアリング結果 + 審査基準 → 事業計画書テンプレートに流し込み

出力形式:
- HTML（プレビュー用）
- DOCX（ダウンロード・編集用）
- PDF（提出用）

事業計画書の構成:
1. 事業概要
2. 現状の課題と解決策
3. 具体的な取り組み内容
4. 実施体制
5. スケジュール
6. 補助事業の効果（数値目標）
7. 資金計画
8. 賃上げ等の加点要素
```

## 審査基準マッピング

各補助金の審査項目と、事業計画書の記載内容を対応させる:

| 審査項目（例: ものづくり補助金） | 事業計画書で書くべきこと |
|------|------|
| 技術面（技術的課題の有無、新規性） | 課題の具体性、解決策の独自性 |
| 事業化面（市場ニーズ、競合優位性） | ターゲット市場分析、差別化ポイント |
| 政策面（地域経済への貢献、雇用） | 雇用計画、地域サプライチェーンへの影響 |
| 経営力面（財務状況、実施体制） | 組織体制図、財務データ |

## 知識ベース（審査基準テンプレート）

### ものづくり補助金
- 審査配点: 技術面(30) + 事業化面(30) + 政策面(20) + 経営力(20)
- 加点: 賃上げ(+10), 事業承継(+5), DX推進(+5), グリーン(+5)
- 重要キーワード: 「革新的」「生産性向上」「付加価値額」

### IT導入補助金
- 審査配点: 事業面(60) + 計画目標値(40)
- ITツール選定理由が重要
- 労働生産性・売上高の数値目標が必須

### 小規模事業者持続化補助金
- 審査配点: 経営計画(50) + 補助事業計画(50)
- 商工会議所の確認書が必要
- 「販路開拓」の具体性が評価ポイント

### 事業再構築補助金
- 審査配点: 補助対象事業の適格性 + 事業化点 + 再構築点 + 政策点
- 「思い切った大胆な事業再構築」の説明
- 市場ニーズと事業の整合性

## API連携

### Cloudflare Workers エンドポイント
```
POST /api/navigator/analyze
  Body: { url: "公募要領URL" }
  → 公募要領を解析して構造化データを返す

POST /api/navigator/generate
  Body: { spec: {...}, answers: {...} }
  → 事業計画書HTMLを生成して返す
```
